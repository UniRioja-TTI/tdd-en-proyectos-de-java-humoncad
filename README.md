# Practica III: TDD en Java
## Taller Transversal I: Programacion y Proceso de Informacion

### Descripcion
Este proyecto consiste en el desarrollo de un sistema de gestion de tareas pendientes (ToDo List) aplicando tecnicas de Desarrollo Guiado por Pruebas (TDD). El enfoque principal es el uso de Maven para la gestion de dependencias y JUnit para la realizacion de pruebas unitarias e integracion.

### Estructura del Proyecto
- pom.xml: Archivo de configuracion de Maven que gestiona las dependencias y el ciclo de vida del proyecto[cite: 18].
- src/main/java: Contiene el codigo fuente, incluyendo las clases ToDo, Repositorio, Servicios y Stubs[cite: 31, 33, 41].
- src/main/resources: Ubicacion para archivos de configuracion y recursos externos del sistema[cite: 25].
- src/test/java: Directorio destinado a las clases de prueba unitaria e integracion[cite: 55, 66].

### Funcionalidades
- Creacion y gestion de tareas con nombre, descripcion y fecha limite[cite: 33, 43].
- Persistencia simulada de datos mediante un Stub de base de datos[cite: 33, 35].
- Sistema de alertas automaticas por correo electronico para tareas vencidas[cite: 45].
- Consulta de tareas pendientes y gestion de una agenda de contactos[cite: 35, 44].
- Verificacion de integridad mediante pruebas automatizadas en JUnit[cite: 73, 79].
